Modèle de Données pour Google Sheet

Ce document décrit la structure requise pour le Google Sheet utilisé par l'application "GestionELS".

Feuille 1: Clients

Objectif: Stocke les informations des clients, leurs remises et les données de parrainage.

Nom de la Colonne Description
EmailClé unique pour chaque client.
Raison SocialeNom de l'entreprise du client.
AdresseAdresse postale du client.
SIRETNuméro SIRET du client.
Type de RemiseEx: "Pourcentage" ou "Montant Fixe".
Valeur RemiseLa valeur numérique de la remise.
Nombre Tournées OffertesNombre de courses gratuites restantes.
Code de ParrainageLe code unique que ce client peut partager.
Code UtiliséLe code de parrainage que ce client a utilisé lors de son inscription.
Crédit ParrainageLe montant en euros que ce client a gagné grâce au parrainage.

Feuille 2: Facturation

Objectif: Journal principal pour toutes les réservations et factures générées.

Nom de la Colonne Description
DateDate et heure de la réservation.
Client (Raison S. Client)Nom du client.
Client (Email)Email du client.
TypeType de course (ex: "Normal", "Samedi").
DétailsDescription de la course (nombre d'arrêts, etc.).
MontantPrix final de la course.
StatutStatut de la réservation (ex: "Confirmée").
ValiderCase à cocher pour inclure dans la prochaine facture.
N° FactureNuméro de la facture une fois générée.
Event IDID de l'événement dans Google Calendar.
ID RéservationID interne unique de la réservation.
Note InterneNotes pour l'administrateur.
Tournée Offerte AppliquéeIndique si une course gratuite a été utilisée.
Type Remise AppliquéeLe type de remise qui a été appliqué.
Valeur Remise AppliquéeLa valeur de la remise appliquée.
Lien NoteLien vers une note externe si nécessaire.
ID PDFID du fichier PDF de la facture dans Google Drive.
